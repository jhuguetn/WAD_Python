Tools.PluginTesting.PluginTester.sanitycheck(args)[source]¶
reguires a folder structure
<root>/Plugins/<some path>/<plugin name> <root>/Testing/<some path>/<plugin name>
The Testing plugin folder should at least one of:
series/config_<id>.xml (plugin configuration file) instance/config_<id>.xml (plugin configuration file) study/config_<id>.xml (plugin configuration file)

In addition there needs to be a <testdata> folder with a folder dicom_<id>, in which test data is present for this plugin.

Example:
plugin: <root>/Plugins/CT/CT_Philips_QuickIQ plugin files: .... testing: <root>/Testing/CT/CT_Philips_QuickIQ testing files: series/config_body.xml,series/config_head.xml testdata: <dataroot>/CT/CT_Philips_QuickIQ/dicom_head/ and <dataroot>/CT/CT_Philips_QuickIQ/dicom_body/
